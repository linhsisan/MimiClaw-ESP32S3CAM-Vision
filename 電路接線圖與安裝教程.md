# 🐾 MimiClaw - 個人化 AI 智慧助理 (ESP32-S3)

MimiClaw 是一個基於 ESP32-S3 與大語言模型 (GPT-4o-mini) 打造的實體 AI 助理。它不僅能透過 Telegram 與你進行自然語言對話，還能控制實體世界的感測器、馬達、全彩燈光，並將對話內容即時顯示在專屬的 TFT 螢幕上。

---

## ✨ 核心特色 (Core Features)

- **🧠 雲端大腦介入**：內建 LLM Proxy，支援 OpenAI API，將自然語言轉化為實體硬體控制指令。
- **📱 Telegram 遠端通訊**：隨時隨地透過手機 Telegram App 與 MimiClaw 傳送文字與指令。
- **💬 視覺化對話螢幕**：透過 ST7789 2.8 吋螢幕，以精美的「氣泡對話框」呈現即時互動。
- **🌍 自動時區記憶**：具備 SNTP 網路自動校時，並支援跨國動態時區切換與 SPIFFS 永久記憶。
- **🌈 智慧情境燈光**：支援 WS2812 千萬色彩控制與平滑的階梯式呼吸燈效果。
- **⚙️ 模組化工具鏈**：AI 大腦可自主呼叫多個硬體工具（如讀取溫濕度、轉動馬達、切換燈光）。

## 📖 進階功能使用手冊

MimiClaw 支援強大的「**多工具鏈式呼叫**」，只要在自然語言中加入「**切換**」二字，就能讓 AI 連續執行多個硬體動作。

### 🌍 1. 自動時區記憶系統 (Timezone Memory)
當你帶著 MimiClaw 跨國旅行時，系統會自動將時區設定永久存入 Flash 記憶體 (`/spiffs/tz.txt`)，開機自動載入當地時間。
* **查詢時間**：「現在幾點？我們在哪個時區？」
* **出國切換**：「我現在已經從台北移動到**日本**了，請幫我把設定**切換**為日本時間，並記住這個設定。」
* **回國恢復**：「我回台灣了，幫我把時間調回台北時區。」

### 🌈 2. WS2812 智慧情境燈與呼吸效果
AI 被嚴格限制使用專用驅動庫來控制燈光，具備防呆機制，並支援獨立亮度控制。
* **基礎開燈**：「幫我把工作燈打開，設定為溫暖的黃光，亮度只要 30% 就好。」
* **動態呼吸燈**：「請幫我啟動**呼吸燈**：先把燈光設定為紅色，亮度 100%。**切換**計時器等待 2 秒，**切換**亮度變 50%，再**切換**等待 2 秒，最後**切換**亮度變 10%。」
* **感測器聯動**：「讀取一次溫濕度，如果溫度超過 30 度，就**切換**把燈光變成紅色 100% 亮度警告我。」

## 【單元一】快閃記憶體 (Flash) 故障排除與深度修復 SOP

在頻繁的開發與燒錄過程中，若終端機不斷印出 `invalid segment length 0xffffffff` 或 `No bootable app partitions` 且設備陷入「無限重新開機」的死迴圈，代表晶片內部的分區表或開機指標（otadata）已經嚴重損毀。

**發生原因：**
通常是因為燒錄中斷（線材鬆脫）、電壓不穩，或是切換了硬體配置檔但未清除舊快取，導致新舊資料在記憶體中重疊衝突。

### 🛠️ 深度修復標準作業流程 (SOP)

遇到此情況，常規的燒錄指令將無法作用，請務必嚴格執行以下「全盤抹除與重置」流程：

**步驟 1：確認設備連接與通訊埠**
* 將 ESP32 設備透過可靠的 USB 傳輸線連接至電腦。
* 開啟 Windows「裝置管理員」，展開「連接埠 (COM 和 LPT)」，確認設備所在的 COM 號碼（例如：`COM11`）。

**步驟 2：執行全盤抹除指令 (Erase Flash)**
* 開啟 ESP-IDF 終端機（ESP-IDF Command Prompt）。
* 切換至專案目錄（`cd C:\mimi_work\mimiclaw`）。
* 輸入以下指令並按下 Enter（請將 `COM11` 替換為實際號碼）：
  ```bash
  idf.py -p COM11 erase-flash
  ```

**步驟 3：物理急救（針對卡在 Connecting... 的設備）**
由於設備正處於無限重啟的死迴圈，電腦往往無法順利連線（畫面會卡在 `Connecting...` 並反覆跳轉）。此時必須進行物理按鍵操作，強制晶片進入「下載模式」：
1. **【按住不放】** 開發板上的 `BOOT` 實體按鍵。
2. **【按一下】** 開發板上的 `RST` (或 EN) 實體按鍵（按一下即可放開）。
3. **【放開】** 剛剛一直按著的 `BOOT` 鍵。
> **✅ 成功特徵：** 完成上述動作後，終端機畫面會瞬間接通，並出現 `Erasing flash (this may take a while)...`，約 3~5 秒後顯示 `Chip erase completed successfully` 即代表洗腦成功。

**步驟 4：重新完整燒錄**
* 記憶體已恢復原廠空白狀態，請輸入以下指令重新將引導程式、分區表與主程式完整燒錄進去：
  ```bash
  idf.py build flash monitor
  ```
*(註：若燒錄時再度卡在 Connecting...，請重複一次步驟 3 的物理按鍵操作即可。)*

## 【單元二】螢幕驅動標準化切換流程 (Menuconfig)

MimiClaw 系統底層已完整內建 ILI9341 與 ST7789 的硬體驅動與自動橋接邏輯。當硬體工程師更換了不同型號的 TFT 螢幕時，**嚴禁手動修改程式碼**，請一律使用系統內建的 Kconfig 圖形化選單進行切換。

> **⚠️ 嚴重警告：**
> 切換螢幕型號後，**絕對不可直接執行 `idf.py flash`**。由於編譯器（Ninja）具有快取機制，若未進行「完全清理 (fullclean)」，系統將沿用舊的驅動快取，導致開機後出現「畫面一半雪花」、「文字側翻 90 度」或「顏色反相」等致命錯誤。

### 🛠️ 螢幕切換標準作業流程 (SOP)

請依序嚴格執行以下五個步驟：

**步驟 1：開啟系統配置選單**
* 在 ESP-IDF 終端機的專案目錄下，輸入以下指令開啟圖形化選單：
  ```bash
  idf.py menuconfig
  ```

**步驟 2：導航至顯示器設定選項**
* 進入選單後，使用鍵盤的 **【上下方向鍵】** 進行移動。
* 找到並進入 **`MimiClaw Display Configuration`** 選項（按 Enter 進入）。
* 進入 **`Select Display Driver Model`** 選項。

**步驟 3：選取對應的硬體型號**
* 根據目前實際連接的螢幕硬體，選擇對應的型號：
  * `ILI9341 (Standard 2.4 inch)`
  * `ST7789 (New 2.4 inch / Higher Brightness)`
* 按下 **【Enter】** 確認選擇。
* 依序按下 **【S】** 鍵儲存設定，再按 **【Enter】** 確認路徑，最後連續按 **【Q】** 鍵退出設定畫面。

**步驟 4：執行深度清理 ⚠️ (核心關鍵步驟)**
* 退出選單後，在終端機輸入以下指令，強制清除所有舊型號的 `.obj` 編譯快取檔案：
  ```bash
  idf.py fullclean
  ```
> *此步驟需等待系統刪除 build 資料夾內容，確保下一次編譯是 100% 重新打包。*

**步驟 5：重新編譯、燒錄與驗證**
* 執行標準的編譯與燒錄指令：
  ```bash
  idf.py build flash monitor
  ```
* **驗證結果：** 燒錄完成並重啟後，請仔細觀察終端機 (Monitor) 的開機日誌。
  * 若切換為 ST7789，日誌中應出現綠色字體的：`[OK] ST7789 DRIVER LOADED: 240x320 PORTRAIT`
  * 若切換為 ILI9341，則不會出現上述 ST7789 專屬日誌。
  * 確認實體螢幕顯示直向滿版，且氣泡顏色與文字無反相，即代表切換大功告成！

在開始點擊任何編譯按鈕之前，請先確保你的「地基」是穩固的。不穩定的硬體或缺漏的軟體會導致後續無止盡的報錯。

  * **核心大腦**：請準備你的 **ESP32-S3** 開發板。
  * **傳輸線材**：準備一條\*\*「具備數據傳輸功能」\*\*的 Type-C 線。*(新手最常踩的終極大坑，就是隨手拿了一條耳機或小家電附贈的「純充電線」，導致電腦永遠抓不到板子！請務必接上手機測試能否傳輸檔案。)*

### 2\. 開發軟體 (ESP-IDF 與編輯器)

  * **ESP-IDF 安裝**：請至樂鑫官網下載 **ESP-IDF Offline Installer (離線安裝包)**。

  * **編輯器安裝**：下載安裝 **Visual Studio Code (VS Code)**。接著在左側擴充功能（Extensions）欄位中，搜尋並安裝官方的 **Espressif IDF** 插件。

## 📖 第四章：新手最大魔王——喚醒 `idf.py` 指令

在進入設定階段前，你必須認識 ESP-IDF 的靈魂指令：**`idf.py`**。
由於 ESP-IDF 為了避免弄亂你的系統環境，它**不會**將自己加入全域變數。如果你隨便打開電腦裡的「命令提示字元 (cmd)」或 VS Code 預設的終端機，並輸入 `idf.py`，電腦一定會無情地回應：**「找不到命令 / Command not found」**。

**如何正確喚醒它？請從以下兩種方式中二選一：**

  * **方式 A (CLI 玩家 - 使用桌面捷徑)**：安裝完 ESP-IDF 後，你的 Windows 桌面或開始選單會產生一個名為 **`ESP-IDF 5.3 PowerShell`** (或 CMD) 的黑盒子圖示。**你只能用這個捷徑來打指令**，它在開啟時會自動執行環境載入指令。
  * **方式 B (VS Code 玩家 - 內建喚醒)**：在 VS Code 裡面，**絕對不要**直接按 `Ctrl + ~` 叫出終端機！請按下鍵盤的 **`F1`**，輸入並選擇 **`ESP-IDF: Open ESP-IDF Terminal`**。此時從底部彈出的終端機，才是聽得懂 `idf.py` 的專屬終端機。

*(⚠️ 以下第五章與第六章提到的「終端機」，皆是指上述這兩種專屬環境！)*

## 📖 第五章：核心觀念——「對齊」ESP-IDF (Alignment)

這是一切成敗的關鍵：**「下載別人的程式碼，如果沒有對齊 ESP-IDF 環境，是絕對無法編譯與燒錄的！」**

原作者上傳的專案裡，可能殘留了舊電腦的絕對路徑、舊的依賴庫版本，甚至不同的晶片設定。我們必須將這些舊痕跡徹底抹除，並與你的實體晶片進行「對頻」。請在專屬終端機中（確認已移動至 `C:\mimi_claw\` 資料夾下）執行以下動作：

1.  **環境大掃除 (清除舊作者路徑)**：
      * 指令：`idf.py fullclean`
      * *說明：這會刪除所有舊的編譯快取，強迫編譯器在你的電腦上從零開始建構新的路徑。*
2.  **晶片對齊 (鎖定目標為 S3)**：
      * 指令：`idf.py set-target esp32s3`
      * *說明：告訴編譯系統我們要使用的是具備 AI 加速與向量運算的 S3 晶片，而非普通的 ESP32。*
3.  **依賴組件對齊 (下載缺漏檔案)**：
      * 指令：`idf.py reconfigure`
      * *說明：這會掃描專案，並自動從網路下載 MimiClaw 依賴的外部函式庫（如 JSON 解析器、LED 驅動）。*
4.  **硬體記憶體對齊 (Menuconfig 設定)**：
      * 指令：`idf.py menuconfig` (開啟藍底白字的圖形化設定選單)。
      * **必檢查項 1 (打開外部大腦 PSRAM)**：進入 `Component config` -\> `ESP PSRAM`，將其設為 **Enabled**。*（MimiClaw 處理大語言模型需要龐大的記憶體，若沒對齊開啟此項，晶片運算時必定崩潰。）*
      * **必檢查項 2 (Flash 分區與容量)**：進入 `Serial flasher config` -\> `Flash size`，請依照你的硬體規格選擇 **8MB** 或 **16MB**。
      * 設定完成後，按下鍵盤 `S` 存檔，按 `Enter` 確認，再按 `Q` 離開選單。

## 📖 第六章：黃金四連擊——編譯與燒錄 (Build & Flash)

對齊完成後，我們終於要把程式碼化為二進制數據，塞進實體晶片中了。請將開發板接上電腦，打開裝置管理員的「連接埠 (COM 和 LPT)」，確認它分配到了哪一個編號（例如 **COM3**）。

這份教科書提供兩種模式，請擇一執行：

### 模式 1：CLI 指令行模式 (駭客推薦，避錯率最高)

只要你接手任何別人的 ESP32 專案，請毫不猶豫地在終端機敲下這套「黃金四連擊」：

```powershell
idf.py fullclean          # 第一擊：掃除殘留
idf.py set-target esp32s3 # 第二擊：對齊晶片
idf.py reconfigure        # 第三擊：載入組件

# 第四擊：指定 COM 埠，並執行編譯(build) + 燒錄(flash) + 監控(monitor)
idf.py -p COM3 flash monitor
```

*(系統會開始跳動大量綠色文字進行編譯，成功後會出現進度條將資料寫入晶片，最後自動打開監控視窗顯示 MimiClaw 的開機日誌。)*

### 模式 2：VS Code 視覺化模式 (直覺操作)

如果你不喜歡打字，VS Code 底部有一排藍色的狀態列為你準備了所有功能：

1.  **對齊檢查**：確認底部狀態列的微控制器顯示為 **`esp32s3`**。
2.  **選取 COM Port**：點擊插頭圖示（`🔌`），選取你的 COM3（或其他正確編號）。
3.  **清除快取**：點擊底部的 **垃圾桶圖示 (Full Clean)**。
4.  **一鍵啟動**：點擊底部的 **火焰圖示 (Build, Flash and Monitor)**。
      * 下方終端機便會開始運作。當進度條跑到 `Writing at 0x00010000... 100% (Hash verified)` 即代表燒錄成功！

## 📖 第七章：燒錄急救箱 (Troubleshooting)

如果你在最後一刻看到了紅色錯誤，或是進度條卡住不動，請深呼吸，對照以下常見的新手症狀：

  * 🚨 **症狀一：燒錄時卡在 `Connecting......._____....._____` 不動，最後 Time out。**
      * **原因**：ESP32 處於自我保護狀態，沒有成功切換到「接收下載」的模式。
      * **終極解法 (手動進入 Boot 模式)**：當畫面上開始出現 `Connecting...` 的字樣時，**按住開發板上的 `BOOT` 按鈕不放** -\> **點擊一下 `RESET` (或標示為 EN) 按鈕** -\> **放開 BOOT 按鈕**。電腦就會瞬間抓到晶片並開始燒錄！
  * 🚨 **症狀二：終端機顯示「無法辨識 'idf.py' 詞彙」**
      * **解法**：這代表你沒有使用專屬的環境。請立刻回頭重看**第四章**。
  * 🚨 **症狀三：編譯出現 `fatal error: mimi_secrets.h: No such file or directory`**
      * **解法**：代表你沒有將靈魂注入。請回頭看**第二章**，去 `main/` 目錄下手動建立密碼檔。
  * 🚨 **症狀四：燒錄明明成功 100%，但監控畫面不斷印出亂碼並無限重新開機 (Guru Meditation Error)**
      * **解法**：這通常代表記憶體溢出。請回頭看**第五章**的 `menuconfig` 步驟，你必定是忘記將 **PSRAM** 設為 Enabled 了。請去打開它並重新點擊燒錄。

*(💡 溫馨提醒：當你處於 Monitor 監控畫面觀察 AI 日誌時，若想退出回到可輸入指令的狀態，請按下鍵盤的 `Ctrl` + `]` )*
